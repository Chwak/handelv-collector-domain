# Collector Domain - Data Requirements (Phase 1)

## Overview
The Collector Domain manages all data related to buyers/collectors, their taste profiles, collecting strategies, saved items, follows, and community influence. This is a **SOURCE domain** that emits events consumed by other domains.

---

## 1. DATA CAPTURED DIRECTLY FROM FRONTEND

### 1.1 Initial Registration (From Auth Flow)
**Source**: Sign-up form → Auth Domain → Collector Domain

```typescript
// Received via SNS event after email verification
{
  userId: string,              // From Cognito (sub)
  email: string,               // From sign-up form
  username: string,            // From sign-up form
  fullName: string,            // Composed from firstName + lastName
  givenName: string,           // From sign-up form
  familyName: string,          // From sign-up form
  phoneNumber?: string,        // Optional from sign-up form
  emailVerified: boolean,      // From Cognito verification
  emailVerifiedAt: string,     // Timestamp
  experience: string[],        // ["maker", "collector"] from sign-up
}
```

**Tables Affected**:
- `collector_profiles_table` - Creates initial profile

**Flow**:
1. User signs up with collector experience selected
2. Auth domain verifies email
3. `UserRegistrationComplete` event published to SNS
4. Collector domain creates profile from event data

### 1.2 Profile Setup (Onboarding)
**Source**: Collector onboarding flow (frontend)

```typescript
// POST /collector/profile/setup
{
  // Display Name (public name shown to makers)
  displayName: string,                  // Required - "Art Enthusiast" or real name
  
  // Location
  location: {
    country: string,                    // Required - "USA"
    state: string,                      // Required - "New York"
    city: string,                       // Required - "Brooklyn"
    zipCode?: string,                   // Optional - "11201"
    timezone: string,                   // Required - "America/New_York"
  },
  
  // Profile
  bio?: string,                         // Optional - "Passionate about handmade ceramics..."
  profileImageUrl?: string,             // Optional - S3 URL after upload
  
  // Privacy Settings
  profileVisibility: 'public' | 'makers_only' | 'private',
  showPurchaseHistory: boolean,
  showCollections: boolean,
  
  // Initial Interests (will expand in taste profile table)
  initialInterests: string[],           // ["ceramics", "woodwork", "jewelry"]
  
  // Budget Range (for personalization)
  typicalBudgetRange: {
    min: number,
    max: number,
  },
}
```

**Tables Affected**:
- `collector_profiles_table` - Updates profile with setup data

**Validation Rules**:
- displayName: 3-50 characters
- bio: max 500 characters
- initialInterests: 1-10 items
- typicalBudgetRange.min < typicalBudgetRange.max

### 1.3 Taste Profile Setup (Onboarding/Settings)
**Source**: Collector dashboard → Taste profile wizard

```typescript
// POST /collector/taste-profile/create
{
  profileVersion: string,               // "v1" (increments with major changes)
  
  // Aesthetic Preferences
  aestheticPreferences: {
    dominantStyles: Array<{
      style: string,                    // "Minimalist", "Rustic", "Modern"
      affinity: number,                 // 0.0-1.0
    }>,
    
    // Color preferences (hex codes with affinity scores)
    colorAffinityScores: {
      [hex: string]: number,            // {"#FFFFFF": 0.9, "#8B4513": 0.7}
    },
    
    texturePreferences: {
      smooth: number,                   // 0-10
      rough: number,                    // 0-10
      organic: number,                  // 0-10
      geometric: number,                // 0-10
    },
    
    scalePreferences: 'miniature' | 'tabletop' | 'statement' | 'monumental' | 'mixed',
    
    patternComplexity: 'minimalist' | 'moderate' | 'intricate' | 'maximalist',
  },
  
  // Functional vs Decorative
  functionalVsDecorative: number,       // 0-1 (0=pure decor, 1=pure function)
  modernVsTraditional: number,          // 0-1
  naturalVsSynthetic: number,           // 0-1
  symmetryVsAsymmetry: number,          // 0-1
  
  // Value Alignment
  valueAlignment: {
    sustainability: number,             // 1-10 (importance)
    localArtisan: number,               // 1-10
    traditionalMethods: number,         // 1-10
    modernInnovation: number,           // 1-10
    storyAndHeritage: number,           // 1-10
    investmentValue: number,            // 1-10
  },
  
  // Budget Profiles
  budgetProfiles: {
    everydayItems: { min: number, max: number },
    specialOccasions: { min: number, max: number },
    investmentPieces: { min: number, max: number },
  },
  
  // Context
  lifestageContext?: string,            // "new_home", "wedding", "retirement"
  styleConfidenceLevel: number,         // 1-10 (how sure they are of their taste)
}
```

**Tables Affected**:
- `collector_taste_profile_table` (NEW) - Deep taste analysis

**Validation Rules**:
- affinity scores: 0.0-1.0
- texture preferences: 0-10
- value alignment: 1-10
- budget min < max

### 1.4 Acquisition Strategy Setup
**Source**: Collector dashboard → Collecting strategy

```typescript
// POST /collector/strategy/setup
{
  strategyPeriod: string,               // "2026-02" (monthly)
  
  // Budget
  monthlyBudget: number,                // Total budget for month
  budgetByCategory: {
    [category: string]: number,         // {"ceramics": 500, "jewelry": 300}
  },
  
  // Goals
  acquisitionGoals: Array<{
    goal: string,                       // "Complete tea set", "One piece per maker"
    priority: number,                   // 1-5
    deadline?: string,                  // ISO date
    progress: number,                   // 0-100%
  }>,
  
  // Collection Gaps
  collectionGaps: Array<{
    category: string,                   // "Serving bowls"
    description: string,                // "Missing large serving bowl"
    targetPrice: number,                // Max willing to pay
    urgency: 'low' | 'medium' | 'high',
  }>,
  
  // Target Makers
  targetMakers: Array<{
    makerId: string,
    reason: string,                     // "Love their glaze technique"
    targetItem?: string,                // "Dinner plates"
    maxPrice: number,
  }>,
  
  // Watch List
  watchList: Array<{
    productId: string,
    currentPrice: number,
    targetPrice: number,                // Will buy at this price
    expiryDate?: string,                // Stop watching after
  }>,
  
  // Strategy
  investmentVsPersonal: number,         // 0-1 ratio
  timeHorizon: 'immediate' | 'quarterly' | 'annual' | 'lifetime',
  
  // Price Tolerance
  priceToleranceByUrgency: {
    low: number,                        // % premium (e.g., 5)
    medium: number,                     // % premium (e.g., 15)
    high: number,                       // % premium (e.g., 30)
  },
  
  // Seasonal
  seasonalBudgetAllocation: {
    q1: number,
    q2: number,
    q3: number,
    q4: number,
  },
  
  // Thresholds
  splurgeThreshold: number,             // Willing to exceed budget by this %
  dealThreshold: number,                // Will buy if discount exceeds this %
  
  // Financial Events
  financialEvents: Array<{
    date: string,
    type: string,                       // "bonus", "tax_refund"
    amount: number,
    allocatedToBuying: number,
  }>,
}
```

**Tables Affected**:
- `collector_acquisition_strategy_table` (NEW) - Strategic collecting

**Validation Rules**:
- monthlyBudget: > 0
- priority: 1-5
- progress: 0-100
- ratios: 0-1
- percentages: 0-100
